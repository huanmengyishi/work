## 2. 临时项目初始化

使用隔离的临时 HOME/XDG，避免触碰真实配置：

```bash
tmp="$(mktemp -d)"
mkdir -p "$tmp/home" "$tmp/config" "$tmp/data" "$tmp/workspace"
cd "$tmp/workspace"
HOME="$tmp/home" \
XDG_CONFIG_HOME="$tmp/config" \
XDG_DATA_HOME="$tmp/data" \
DEEPSEEK_API_KEY= \
~/AI-Agent/.venv/bin/agent init
```

预期：`.project-agent` 初始化成功；`secrets.env` 权限为 0600。完成后删除临时目录，不要复制任何真实配置。

## 3. P0-1 修复后的无 Key/无下载回归

前置：临时环境中可以导入 Chroma，但没有 Key，也没有模型缓存。

```text
执行一个普通自然语言任务
-> 2 秒内明确报告缺少 DEEPSEEK_API_KEY
-> 退出码为 1
-> 网络访问次数为 0
-> 不出现 onnx_models 下载
-> 不创建 Chroma embedding 模型缓存
```

自动化测试应 mock 网络和 embedding 初始化；任何调用都立即失败测试，不能依赖断网超时。

## 4. P0-2 修复后的退出码契约

```text
completed Session          -> 0
failed/resumable Session   -> 2
不可恢复配置/运行错误       -> 1
Ctrl+C                     -> 130
```

Queue、Daemon、直接任务和 REPL 的文档必须说明各自语义。REPL 正常 `/exit` 仍为 0。

## 5. PTY 检查

在真实 PTY 中逐项检查：

1. 启动后出现项目名、首次使用提示和 Prompt。
2. 空 Enter 显示“未输入请求”。
3. 输入普通任务后立即显示正在处理状态。
4. `Ctrl+C` 返回可恢复状态，不退出整个 REPL。
5. `/exit` 正常退出。
6. 40 列和 25 列终端中，CJK、组合字符、旗帜、肤色、Keycap 和 ZWJ emoji 不被截断为半个 grapheme。

## 7. 在线验收边界

只有离线检查全绿并且用户明确同意 API 成本后，才执行：

```bash
agent doctor --online
```

随后按“文本小案例 -> Word 案例 -> 一次大型案例”的顺序验收。大型案例必须预先固定：

- `git_head` 与 dirty 状态
- Prompt/fixture/runner 哈希
- 最大逻辑请求数
- 最大总 tokens
- 最大 wall time
- 成功语义门
- 失败后停止重跑的规则

验收记录不得包含 Key、Cookie、Token、Session 原文、Memory、PTY 原始捕获或私有工具附件。
