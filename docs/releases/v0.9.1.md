# Deep Agent v0.9.1

Release date: 2026-07-14

## Summary

v0.9.1 stabilizes the core interfaces ahead of v1.0. It makes ContextBuilder
the only model-context entry, freezes executable contracts for the core chain,
adds a minimal versioned Event Bus contract, validates AgentState, and makes
DeepSeek routing cost-aware without adding another provider.

## Changes

- PromptBuilder now accepts only `ContextPackage` through `build_initial()` and
  `build_resume()`. The old distributed context parameters and
  `append_resume()` entry are removed.
- Added `agent/contracts.py` and interface contract tests for CLI, Runtime,
  AgentState, Prompt, ToolRequest/ToolResult, Capability, and Permission order.
- Added stable Event serialization, `run_id`, subscribe/unsubscribe, existing
  Event publication, timestamp validation, JSONL schema parity, nested publish
  behavior, and subscriber failure isolation.
- Added AgentState schema/frozen-field constants and validation for identity,
  plan DAGs, routes, Context manifests, timestamps, and future schemas.
- Added deterministic `cost_class` routing: low, balanced, or high. DeepSeek
  remains the only model provider.
- Added `TaskPlanFactory`. Runtime now uses TaskRouter as the only classifier;
  `TaskStrategySelector` is deprecated and retained only as a compatibility
  facade.
- Added a detailed architecture and Event Bus extension guide.

## Compatibility

- Existing v0.8/v0.9 Session schema v1 records remain loadable and upgrade on
  Resume.
- Existing saved Model routes without `cost_class` infer it from their tier.
- External callers using PromptBuilder's removed v0.8 distributed parameters
  must first create a `ContextPackage`.
- No configuration migration is required. Existing values are preserved.

## Explicitly not included

- full Event migration;
- durable tool Intent Journal;
- GitHub/Jira/Notion integrations;
- multi-Agent or Worker Runtime;
- non-DeepSeek providers;
- large Runtime redesign.

## Validation

- 130 pytest tests passed, including real PTY coverage inherited from the full
  suite;
- Ruff check passed;
- Ruff format check passed;
- compileall passed;
- `git diff --check` passed;
- GitHub Actions continues to use `actions/checkout@v5` and
  `actions/setup-python@v6`, which run on the current Node.js runtime rather
  than deprecated Node.js 20 actions.

## Rollback

Use tag `v0.9.0` and reinstall the editable package. v0.9.1 adds no destructive
configuration migration and does not delete Session, Memory, or project data.
