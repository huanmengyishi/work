# Deep Agent v0.10.0

Release date: 2026-07-14

## Summary

v0.10.0 completes the Event Bus migration of Runtime automatic side effects
while preserving the frozen CLI/Runtime/State/Prompt/Capability/Permission
boundaries and the DeepSeek-only model policy.

## Added and changed

- Added required/best-effort subscriber metadata, named deliveries,
  `EventDispatch`, and `dispatch_required()` fail-closed semantics.
- Centralized automatic subscribers in `RuntimeEventPipelines`.
- Migrated Session checkpoint/finalize to required events with terminal ordering
  that prevents derived events from an unpersisted final state.
- Distinguished Session-owner commit from later required-observer failure so
  recovery never finalizes from an uncertain checkpoint.
- Migrated Context Memory usage to a required, SQLite-transactional,
  idempotent event keyed by run/evidence.
- Kept automatic Memory/Reflection terminal processing idempotent by `run_id`.
- Migrated capability health from ToolManager direct writes to a best-effort
  `tool.finished` subscriber without changing permission order or ToolResult.
- Replaced general payload logging with a bounded Audit allow-list that excludes
  Prompt, reasoning, messages, AgentState, tool arguments/output, and secrets.
- Added private, bounded per-project aggregate metrics.
- Mapped untrusted project IDs to stable safe storage keys for Metrics, Health,
  Daemon, and Parallel paths while preserving normal historical IDs.
- Migrated visible Thinking/progress to a best-effort UI event; reasoning chunks
  remain visible but are not written to Audit or Metrics.
- Added `events.metrics_enabled: true` as an add-only default.

## Compatibility and limits

- DeepSeek remains the only provider.
- Event schema v1 and the v0.9.1 public interface contract remain unchanged.
- Existing Session, Memory, and project data are preserved.
- The Memory database gains an additive `memory_usage_events` table.
- The Event Bus remains synchronous and process-local. There is no replay,
  cross-process broker, Worker Runtime, multi-Agent system, or Durable Intent
  Journal in this release.

## Verification

- 173 pytest cases passed, including required-owner failure injection,
  checkpoint/finalize ordering, duplicate Memory events, Resume, Health failure
  isolation, Audit/Metrics privacy and malformed data, and real PTY Thinking.
- Ruff check passed.
- Ruff format check for `agent` and `tests` passed.
- compileall and `git diff --check` passed.
- GitHub Actions uses `actions/checkout@v5` and `actions/setup-python@v6` for
  Python 3.11/3.12/3.13, avoiding the Node.js 20 runner deprecation.

## Rollback

```bash
cd ~/AI-Agent
git switch --detach v0.9.1
.venv/bin/pip install -e .
```

Return with `git switch main`. v0.9.1 safely ignores the additive SQLite table
and metrics file. Rollback does not delete Session, Memory, or project data.
