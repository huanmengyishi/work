# Deep Agent v0.11.0

Release date: 2026-07-15

## Scope and Compatibility Boundary

Version 0.11.0 replaces the fixed-round failure path with a bounded Agent loop
that can continue through tool batches, reserve space for a final answer, and
fail with resumable evidence when completion is not verified. It addresses long
document/repository work, oversized tool output, repeated exploration,
interrupted tool batches, terminal width, network retry, and inaccurate
completion claims as one reliability change rather than isolated patches.

DeepSeek remains the only model provider. The architecture remains:

```text
CLI -> Runtime -> AgentState -> Prompt -> Capability -> Permission
ToolRequest -> PermissionManager -> ToolResult
```

The package version is `0.11.0` and the current AgentState schema is 6. No
multi-provider router, worker Runtime, plugin framework, or permission bypass was
introduced.

## Agent Loop and Termination

- The 4/8/16/24 mode value is a soft tool-turn target. The default hard limit is
  32 tool turns.
- A tool turn advances only after every retained call from one assistant
  response has one ordered result and the complete batch is checkpointed.
  Context compaction, output continuation, corrective responses, and final
  synthesis are model requests but not tool turns.
- Tool execution may stop at the soft target only after the unified execution
  gate has the required Task Graph, real non-plan tool, explicit
  single-validation, and requested-artifact evidence. Missing evidence is
  reported together and continues toward the hard limit.
- One model response retains at most 64 normalized tool calls. At most 16 are
  executable by default; retained calls after the first 16 receive paired
  Runtime-denied results and are not silently dropped.
- `stop`, `tool_calls`, and an empty/missing finish reason are usable.
  `content_filter` or an unknown finish reason causes zero execution of any
  accompanying calls. Runtime asks once for a protocol-valid replacement, then
  fails with an exact Resume command if it repeats.
- `finish_reason=length` discards incomplete tool-call JSON and permits at most
  two tool-free continuations by default. Continued requests stay in the same
  request phase and do not consume tool turns.
- When tool execution closes, an independent tool-free final-synthesis phase
  prepares the user answer. Its length continuations are counted separately.
  Tool calls or DSML tool-call text in that phase are rejected without
  execution. In the main loop, structured calls accompanied by DSML protocol
  text also discard the complete response batch with zero tool execution.
- Completion still requires a substantive answer, all required plan steps,
  managed-write evidence for requested artifacts, and re-open verification for
  Word artifacts. Only `implement` in a conditional-mutation plan may be
  skipped; scope, inspection, and verification cannot be skipped.
- TaskRoute schema 2 keeps at most 32 sanitized file and directory hints.
  Explicit directory names must match managed `make_dir` evidence. File and
  Word evidence is replayed by path, so a later delete or undo invalidates an
  earlier apply; undoing that delete can restore the prior active artifact.
- Artifact intent is evaluated per bounded clause with local negation, so
  "do not output credentials" and ignored generated files cannot create a
  false managed-write gate or suppress a separate explicit report request.
  Longer no-evidence/skip-implementation clauses retain conditional-mutation
  semantics, while unconditional repairs remain non-skippable.

The soft/hard split, defensive finish-reason rules, independent final synthesis,
and plan completion gate are local v0.11.0 designs verified in this repository.
They are not claimed as native behavior copied from the reference project.

## Request Budget, Compaction, and Circuit Breaking

With the default convergence configuration enabled, every model request passes
the same bounded preparation path:

1. Each model-visible `ToolResult` is valid bounded JSON with success,
   head/tail stdout and stderr, safe data, original length, and SHA-256 evidence
   when space permits.
2. All results produced by one assistant response share a second aggregate
   limit, so a large parallel batch cannot multiply context without bound.
3. Tool calls and results are repaired into one-to-one contiguous pairs in the
   model-visible projection. Duplicate, orphaned, missing, and out-of-order IDs
   are never sent as a broken protocol sequence.
4. Complete-history compaction reduces old/largest results to previews, then
   stable metadata, then deterministic per-result allocations. If minimum
   records cannot fit, oldest complete call/result groups are replaced by one
   bounded evidence record; half pairs are not retained.
5. Request estimation includes serialized messages, active tool schemas, the
   configured output reserve, and a safety buffer.
6. Above the proactive trigger, only complete old API rounds outside the recent
   protected tail may be summarized. The same configured DeepSeek model is used
   with tools and Thinking disabled.
7. If semantic compaction fails repeatedly, a deterministic AgentState/evidence
   projection replaces it. A request that still exceeds the hard input limit is
   rejected before transmission.

A tool-history compaction exception applies the deterministic hard limit from
the first failure; an oversized raw result is never allowed through. Three
consecutive failures open the circuit that stops retrying the richer path.
The semantic-compaction failure counter and circuit are persisted in AgentState
and survive Resume until a successful compaction resets them.

Typed provider context overflow has two additional bounded stages: cheap
deterministic collapse, then semantic compaction with deterministic fallback.
Each retry must reduce the request; a third overflow fails rather than looping.
If the semantic-compaction circuit is already open, the overflow path makes no
new compaction model request and goes directly to deterministic fallback.
Disabling `auto_compaction_enabled` prevents proactive DeepSeek summarization,
but provider-reported overflow retains its separate bounded semantic recovery
stage. That switch does not disable output reservation or the hard request
limit. Disabling convergence also disables nudges, same-round/history
micro-compaction, and proactive summarization; per-result limits, private
attachment quotas, request budgeting, and emergency hard projection remain
active.

## Tool Orchestration and Interruption Pairing

Only consecutive calls whose capability is active, explicitly marked
`concurrency_safe=True`, restricted to exactly `read` permission, free of a
confirmation gate, and not a stateful read may overlap. The default maximum is
four. Unmarked reads, writes, execution, network access, confirmations,
disabled/unavailable capabilities, Runtime denials, and `file.diff` are serial
barriers. Results are returned in original model-call order and all execution
still passes through Permission Manager.

On `KeyboardInterrupt`, cancellation, or another `BaseException`, already
completed results are retained. Running, cancelled, and not-started calls
receive synthetic failed results with their original request IDs. Runtime then
checkpoints the complete contiguous pair set, marks the Session interrupted,
and re-raises the original exception. Subprocess tools use bounded head/tail
capture and terminate the complete child process group on timeout or interrupt.

## Private Tool-Result Attachments

Results larger than 12,000 serialized bytes are persisted under the current
`.project-agent/tool-results/<session-id>/` directory. The model and AgentState
receive a bounded preview plus attachment metadata; they do not retain an
unlimited raw body. Defaults are:

- 8 MiB hard limit per attachment;
- 12,000-character preview;
- 32,000-character ordinary `tool_result_read` limit;
- 512 attachments and 256 MiB total per Session.

The store validates Session and request IDs, private permissions, write-once
request IDs, regular-file type, symlinks, path containment, count, and total
bytes; reads return SHA-256 metadata. Source-level or attachment-level
truncation is recorded explicitly. A persistence failure does not rewrite an
already successful tool side effect as failed: Runtime keeps the true success,
bounded head/tail, SHA-256, and `attachment_persistence_error`. Results above
the 8 MiB hard limit or already truncated upstream do not claim that a complete
body is available. Attachment directories are Git-ignored and must not be
published.

The hard convergence phase does not reopen ordinary exploration attachments.
Only while `implement` or `verify` is active, at most two reads of at most 12,000
characters may target an attachment produced in the same Session by tests,
diagnostics, document verification, staged-diff verification, or a
non-exploratory shell validation. Unknown, oversized, exhausted, and ordinary
exploration attachments are denied with paired results.

## Convergence and Validation Selection

Large/deep tasks count consecutive read-only rounds and repeated targets. The
soft boundary removes broad discovery; the continuous-read or reserved-round
hard boundary closes targeted reads and Shell/Python file-exploration aliases.
Advertised aliases are resolved to canonical capabilities before enforcement.
The implementation step may use at most two exact, previously read file ranges
of at most 200 lines by default; verification-phase source exploration stays
closed.

`run_tests framework=auto` now checks language manifests before falling back to
a generic `tests/` directory: Python project markers, `package.json`, Cargo, Go,
Gradle, and Maven are considered in that order. npm validation runs only an
existing allowlisted script, preferring `test`, `typecheck`, `check`, `lint`,
then `build`. A failed validation remains honest failed evidence; the completion
gate does not rename an executed failure as a pass.

If the user explicitly requests only one validation attempt, TaskRouter records
`single-validation`. After the first executed test, diagnostic, or recognized
validation shell command, Runtime removes equivalent validation schemas and
denies shell/LSP/test substitutions. A missing dependency or baseline failure
is the one allowed attempt and must be reported, not probed repeatedly.
Runtime-denied or handler-preflight `not_executed` results do not consume the
attempt. Validation wrappers such as `uv run`, `npm --prefix`, `timeout`, and
`python -I -m pytest` are recognized; a compound command containing multiple
validations is denied before shell execution. In hard phase, value forms of
write/fix flags such as `--write=true`, `--apply=yes`, `--update=...`,
`--bless=...`, `--accept=...`, `-wfoo`, and `--install-types` stay closed.

## AgentState Schema 6 and Resume

Schema 6 stores convergence state, model metrics, bounded tool evidence and
attachment metadata, plus total, main-loop, context-compaction, and
final-synthesis request counts. Resume preserves the original objective and
frozen project/Session identity, routes, plan, durable target set,
semantic-compaction breaker, and bounded evidence. It increments `turn`, resets
turn metrics, and reopens per-turn read/stall allowances so recovery is usable.

Resume is checkpoint continuation, not complete transaction replay. This
release has no Durable Intent Journal, cross-process exactly-once guarantee, or
automatic proof of what an external side effect did between execution and its
checkpoint. The user must inspect uncertain external state before retrying.

v0.11.0 can normalize supported older AgentState records on Resume. v0.10.0
cannot directly load or Resume a schema 6 Session; it correctly rejects that
record as a future schema.

## Terminal, Read Output, and Network Reliability

- The single-line progress renderer crops by Unicode grapheme cluster and terminal display
  cells. CJK, combining marks, flags, emoji modifiers, keycaps, and ZWJ
  sequences are not split on a narrow PTY. This does not claim the reference
  project's full-screen Ink layout. It uses the new required `regex` and
  `wcwidth` dependencies; PyYAML remains required.
- Streamed Thinking is kept separate from the carriage-return spinner, so timer
  refreshes do not erase reasoning.
- `read_file` renders `{line_number:>6}→{source}`, for example
  `     1→export const value = 1`. The arrow is an output boundary, not source
  content, and keeps display padding distinct from indentation copied into an
  edit.
- TLS record-layer failures, connection resets, incomplete reads, timeouts, and
  retryable HTTP 408/5xx responses receive bounded same-key retry. HTTP
  401/403/429 rotates to the next key.
- A stream may retry only before a valid reasoning, content, or tool-call delta.
  After any valid delta it is not replayed; Runtime saves a resumable Session to
  avoid duplicating in-flight work.
- `agent resume` without a continuation enters the selected Session REPL. A
  missing Session reports one clean error without a traceback.

## Reference Decision

The comparison used the fixed `claude`-branch snapshot of
`https://gitee.com/free/claude-code` at commit
`b17913e26fd4278ad5cd4b32ed3bde86bf1444e9`, tree
`68ef0259999328e531cc23c81fc80e81cbdabecb`.

| Area | Reference influence | Deep Agent v0.11.0 boundary |
| --- | --- | --- |
| Tool results | Session-scoped local offloading and model-visible previews | Session-private store with local permission/path/quota checks; AgentState keeps preview/metadata |
| Concurrency | Explicit opt-in safety | Strict pure-read/no-confirmation checks; all other capabilities are serial barriers |
| Interruption | Preserve call/result protocol pairing | Synthetic failed pairs, checkpoint, then re-raise the original interruption |
| Display | Grapheme/display-width handling | `regex` + `wcwidth` for the existing line-oriented WSL CLI |
| Context | Micro-compaction and bounded recovery | DeepSeek-only summary path plus deterministic AgentState projection |
| Termination | No direct implementation claim | Local soft/hard turns, finish defenses, and final synthesis |

The reference's multi-provider architecture, full-screen TUI, plugin ecosystem,
and permission assumptions were not added. The implementation was written for
this repository and preserves its single-DeepSeek and centralized Permission
Manager boundaries. See `docs/v0.11.0-end-to-end-gap-analysis.md` for the full
source-path matrix and explicit remaining differences.

## Verification Status

The frozen v0.11.0 candidate collected and passed 454 offline tests. Coverage
includes unit/integration protocol checks and real PTY tests for tool-result
budgets and attachments, pair repair, explicit concurrency, interruption,
soft/hard termination, finish reasons, length continuation, compaction and
circuit fallback, bounded validation reads, manifest-first test selection,
schema-6 Resume, terminal grapheme width, `read_file` arrow output, network
retry, Thinking, and Word render/apply/re-open rules.

The CLI/Console subset contains 18 passing tests. Real PTYs cover 40-column
Readline Enter/help/exit, visible Thinking and interruption recovery, and a
25-column progress line containing flag, skin-tone modifier, keycap, ZWJ, and
combining clusters. A clean Python 3.14 environment installed
`.[dev,browser,semantic]`, passed `pip check`, and passed the same 454 tests.
The smaller `.[dev]` extra installs successfully, but the complete suite's
browser and semantic-index integrations intentionally require their optional
extras.

```bash
.venv/bin/python -m pytest -q
.venv/bin/ruff check agent tests scripts
.venv/bin/ruff format --check agent tests scripts
.venv/bin/python -m compileall -q agent tests scripts
git diff --check
```

The online Word and plain-text cases passed. The last pre-fix large TypeScript
case made 36 logical requests (31 completed tool turns), used 1,477,342 reported
tokens, generated a complete evidence-based report, and passed every semantic
check except `session_completed`. The root cause was deterministic: artifact
intent crossed a clause boundary into “must not output credentials”, while the
long conditional no-change statement was missed. The exact prompt pattern now
passes a deterministic 4-main-loop + 1-final-synthesis completion regression
without `file_apply`, managed-write failure, or Resume output. It was not rerun
online to avoid another high-token charge, so this note does not claim a
post-fix online large-case pass. No API key, private Session, Memory, attachment,
cache, PTY capture, or runtime log belongs in the release.

## Rollback

The previous code tag is `v0.10.0`:

```bash
git switch --detach v0.10.0
.venv/bin/pip install -e .
```

No configuration or private data must be deleted. Do not delete Memory,
`.project-agent`, Sessions, or attachments during rollback testing. Because
v0.10.0 cannot Resume AgentState schema 6, first finish the Session with
v0.11.0, or preserve it and start a separate v0.10.0 Session. Return with
`git switch main`.
