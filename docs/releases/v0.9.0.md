# Deep Agent v0.9.0

Release date: 2026-07-13

## Goal

Make one Agent Runtime choose an execution policy, DeepSeek capability tier,
and bounded context deterministically before the model request. This release
does not add another provider or a set of autonomous Agents.

## Added

- `TaskRouter` classifies task type, scale, risk, and
  simple/standard/large/deep execution mode without an extra API call.
- `ModelRouter` maps the Task route to fast, standard, or deep while enforcing
  DeepSeek as the only provider.
- `ContextBuilder.build_package()` unifies task, project instructions, project
  summary, Session, semantic, Memory, recovery, and capability-summary context.
- Context Package accounting includes rendered headings and separators and
  records omitted or truncated sections in `AgentState.context_manifest`.
- Only Memory records actually included in a Package are marked as used.
- Resume rebuilds a compact Package and monotonically preserves or upgrades
  both the Task route and the exact DeepSeek Model route.
- `.github/workflows/test.yml` defines Ruff, pytest, and compileall checks for
  Python 3.11, 3.12, and 3.13.

## Routing and Thinking

With automatic routing and adaptive thinking enabled:

```text
simple/low-risk -> fast     -> thinking off
normal work     -> standard -> high reasoning effort
deep/high-risk  -> deep     -> max reasoning effort
```

Architecture/refactor work, high-risk mutations, and repeated failures can
promote a route. A brief Resume request cannot downgrade an existing Session.

All tier-specific model settings default to `null`. Fast, standard, and deep
therefore all use `model.model` unless valid DeepSeek model names are explicitly
configured:

```yaml
model:
  provider: deepseek
  model: deepseek-v4-pro
  routing:
    enabled: true
    tier: auto
    fast_model: null
    standard_model: null
    deep_model: null
```

## Context Limits

```yaml
context:
  max_user_request_chars: 32000
  package_limits:
    simple: 12000
    standard: 32000
    large: 48000
    deep: 64000
  max_package_chars_hard_limit: 96000
  max_recovery_context_chars: 6000

runtime:
  max_user_request_chars: 250000
```

The budget covers the bounded user request and fully rendered Context Package
sections. Oversized requests retain bounded head/tail content. The fixed system
prompt, active tool JSON schemas, and later `ToolResult` messages are outside
this character count and retain their separate limits. Private Memory, Session,
and recovery sections are not written to
`.project-agent/cache/context.generated.md`.
Input above the runtime ceiling is rejected with guidance to use a project file
for complete chunked processing.

Configuration migration is add-only: missing defaults are added without
overwriting existing values.
