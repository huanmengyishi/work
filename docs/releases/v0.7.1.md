# Deep Agent v0.7.1

## Release reason

Fix a WSL/GNU Readline interactive-input defect where raw ANSI color sequences
inside the prompt were counted as visible characters. In narrow or wrapped
terminals this could desynchronize cursor placement and make Enter appear not to
submit the request.

## Fix

- Wrap ANSI start/end sequences with GNU Readline non-printing delimiters
  (`\001` and `\002`) when readline is active.
- Flush stdout before waiting for input so the prompt is visible before the
  terminal reads a line.
- Give empty Enter explicit guidance instead of silently re-prompting.
- Print a processing status as soon as a natural-language request is submitted.
- Handle `Ctrl+C` during a task by returning to the REPL and retaining the
  checkpointed Session for `/resume`.
- Require valid Git metadata (`.git/HEAD` or a worktree `gitdir:` file) before
  selecting an ancestor as the project root, avoiding false roots from empty
  `.git` placeholder directories.

## Verification

- Added console regression tests for readline-safe and non-readline ANSI prompts.
- Added a Project Manager regression test for an empty ancestor `.git` directory.
- Verified `/help`, empty Enter, and `/exit` through a real pseudo-terminal.
- Ran the complete test suite, Ruff, formatting, and compile checks.

## Follow-up direction

Keep terminal UI dependency-light. If line editing requirements outgrow GNU
Readline, evaluate a dedicated prompt library behind the existing `ConsoleUI`
boundary rather than introducing terminal behavior into Runtime or CLI dispatch.
