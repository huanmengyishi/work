# Deep Agent v0.6.0

## Release Reason

This release freezes the decision-intelligence layer as a separate rollback
point before language servers and background processes add external runtime
dependencies. The core improvement is one serializable decision model shared by
interactive tasks, resume, queues, and worktree parallel execution.

## Included

- Dependency-aware Task Graph in `AgentState.plan`.
- Step descriptions, dependencies, retry counters, parallel permission, and
  completion criteria with cycle and readiness validation.
- `.project-agent/workspace_memory.json` with detected project facts and a
  preserved manual override section.
- Rule Reflection and optional DeepSeek smart Reflection.
- Serializable Execution Context containing directory, branch, modified files,
  current plan/queue, recent tool/error, snapshot, and prompt phase.
- Persisted Capability Health with Available, Unavailable, Need Config,
  Disabled, and Broken states.
- Model schemas exclude Broken capabilities until health is reset or recovered.
- Queue and parallel Sessions consume the same Task Graph structure.

## Verification

- 44 automated tests passed.
- Ruff lint and format checks passed.
- Python compileall passed.
- Legacy v0.5 Session/Plan JSON remains loadable.
