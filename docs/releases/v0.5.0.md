# Deep Agent v0.5.0

## Release Reason

This release establishes the first GitHub baseline after the runtime, safety,
memory, external-tool, and isolated-execution foundations were verified as one
coherent system. Publishing this point separately provides a stable rollback
anchor before planner and project-intelligence state become more complex.

## Included

- Project-centric DeepSeek runtime and resumable sessions.
- Safe file preview/apply/undo with snapshots and conflict protection.
- SQLite FTS and Chroma memory, Correction learning, and failure recovery.
- MCP stdio, Streamable HTTP, legacy SSE, Tools, and Resources.
- Restricted HTTP, persistent Playwright sessions, OCR, Docker, and Git tools.
- Optional Tree-sitter semantic sidecar index.
- Persistent task queues and threshold-gated Git worktree parallel execution.
- Safe, auto-approve, YOLO, and SUPER YOLO approval levels.

## Verification

- 40 automated tests passed.
- Ruff lint and format checks passed.
- Python compileall passed.
- DeepSeek online Key-pool check passed with all configured entries ready.
- Sensitive-file and credential-pattern scan found no committed secrets.

## Next Optimization Direction

Version `0.6.0` will focus on decision intelligence rather than adding more
tools: a dependency-aware Task Graph, project Workspace Memory, Reflection,
execution-site recovery, and unified Capability Health. Queue and parallel
execution will consume the same Task Graph instead of maintaining separate task
semantics.
